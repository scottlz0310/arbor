SPECIFICATION & INTERACTIVE MOCKUP

Arbor
Git Management Dashboard

複数のローカルgitリポジトリをネイティブデスクトップから一元管理する Tauri v2 アプリケーション。dsx CLI（Go製・既存ツール）をアクション実行層として内包し、git2 で可視化・GitHub API で PR/Issue 連携・Ollama qwen3.5:latest で AI Insight を提供する。

Tauri v2 React 19 + TypeScript Rust / git2 dsx CLI (Go) GitHub REST API v3 Ollama / qwen3.5:latest Windows 11 / macOS 14+
01 プロジェクト概要
CONCEPT

既存CLIツール dsx（環境変数注入・システム更新・リポジトリ更新）をアクション実行層として内包し、GUIによる可視化・意思決定・AI Insight を上位レイヤーとして重ねる。git操作の実装ゼロで、受け入れ側はコマンド実行のラッピングだけに集中できる。

CORE FEATURES
Multi-repo dashboardBranch cleanupCommit graphGitHub PR/IssueFetch / Push / PullStash managergit gc wizardAI Insight Enginedsx 統合
NON-GOALS (v0.1)
— diff ビューアー
— マージコンフリクト解消UI
— GitLab / Bitbucket 対応
— SSH 鍵管理
— CI/CD トリガー
DESIGN PRINCIPLES
— 破壊的操作は必ず確認ダイアログを挟む
— オフライン対応（ローカルgit操作は常に動く）
— 設定は ~/.config/arbor/config.toml で管理
— GitHub PAT は OS キーチェーンに保存（DPAPI / Keychain）
— Rust側は Result<T, String> でエラーを返却
— AI（Ollama）は補助レイヤー。未起動時はルールベースのみで完全動作
— AIにコマンド実行は一切させない（Explain / Prioritize / Risk のみ）
TARGET ENVIRONMENT
Primary  — Windows 11 (x64) / macOS 14+ (arm64)
Rust     — 1.78+ (MSRV)
Node     — 20 LTS (ビルドのみ)
Git      — 2.39+ (libgit2経由、不要)
Screen   — 1280px+ を推奨

RUST — dsx 呼び出しパターン
#[tauri::command]
async fn repo_update() -> Result<DsxOutput, String> {
  let out = Command::new("dsx")
    .args(["repo", "update", "--no-tui", "--jobs", "4"])
    .output().await
    .map_err(|e| e.to_string())?;
  if out.status.success() {
    Ok(parse_dsx_output(&out.stdout))
  } else {
    Err(String::from_utf8_lossy(&out.stderr).to_string())
  }
}
TUI出力は Arbor 側でハンドリングするため --no-tui を付与する。進捗はArbor独自のUIで表示する。
dsx が PATH 上に存在するかを起動時に which dsx / dsx --version で確認し、未インストール時は設定画面にインストール手順を表示する。

05 Tauri Command API 仕様
全コマンドは #[tauri::command] で定義。フロントからは invoke('command_name', args) で非同期呼び出し。エラーは Result<T, String> で返却し、フロント側でトーストに表示する。
list_repositories
() → Vec<RepoInfo>
config.toml 登録済みリポジトリの一覧と状態（ahead / behind / modified）を返す
get_branches
(repo_path: String) → Vec<BranchInfo>
指定リポジトリの全ローカルブランチを取得。merged / ahead / behind / last_commit を付与
delete_branches
(repo_path: String, names: Vec<String>) → Vec<DeleteResult>
複数ブランチを一括削除。current ブランチはスキップ。各ブランチの成否をまとめて返却
get_commit_graph
(repo_path: String, limit: u32) → Vec<CommitNode>
コミット履歴をグラフ描画用データ構造で返す。各ノードに parent_oids / branch_refs / lane（列番号）を付与
fetch_all
(repo_path: String) → FetchResult
全リモートをフェッチ。SSH / HTTPS 両対応（git2 credential callback）
get_pull_requests
(owner: String, repo: String) → Vec<PullRequest>
GitHub REST API GET /repos/{owner}/{repo}/pulls を呼び出し。PAT は keyring から取得
get_issues
(owner: String, repo: String, state: String) → Vec<Issue>
GitHub Issues を取得。state は "open" | "closed" | "all"
run_git_gc
(repo_path: String) → GcResult
git gc --auto + git remote prune origin を実行。解放バイト数を返す

07 AI Insight Engine
PIPELINE
git2ローカルgit状態
+

+

AI の責務（3つのみ）
① Explain — 意味付け
ブランチやリポジトリの状態に「なぜそれが問題か」を日本語で付与する
"type": "explain", "target": "experiment/multi-ai", "reason": "2週間更新されておらず
mainから8コミット遅れている", "source": "ai"
② Prioritize — 優先順位付け
複数リポジトリ・ブランチの中から「今すぐ対処すべき」ものを選別する
"type": "prioritize", "target": "NeuralVenture", "priority": "high", "reason": "更新頻度が最も高い
リポジトリ", "source": "ai"
③ Risk — リスク評価
削除などの破壊的操作に対して、リスクレベルと根拠を事前に提示する
"type": "risk", "action": "delete_branch", "risk": "medium", "reason": "未マージだが
90日間更新なし", "source": "ai"
TYPESCRIPT — Insight 型定義
type InsightType =
  "explain" | "prioritize" | "risk"

type InsightSource = "rule" | "ai"

interface Insight {
  type: InsightType
  target: string
  priority: "low" | "medium" | "high"
  reason: string
  source: InsightSource
  risk?: "low" | "medium" | "high"
}
HYBRID MERGE — ルール + AI
// ルールベース（高速・確定）
const ruleInsights = ruleEngine(state)

// AIベース（非同期・バックグラウンド）
const aiInsights = await aiEngine(state)

// マージして優先度でソート
const insights = [
  ...ruleInsights,
  ...aiInsights
].sort((a, b) =>
  score(b) - score(a)
)
